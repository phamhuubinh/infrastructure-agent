# Mục tiêu hiện tại
Hoàn thành MVP đầu tiên của AI Agent.
Ưu tiên:
1. Hoàn thiện Reasoning Loop.
2. Hoàn thiện KnowledgeTool.
3. Hoàn thiện Child Tools.
4. Hoàn thiện Provider (khi cần).
5. Giữ kiến trúc đơn giản.
---
# Kiến trúc hiện tại
```
User
    ↓
Reasoning Model
    ↓
Agent
    ↓
KnowledgeTool
    ↓
Child Tool
    ↓
Provider (optional)
    ↓
Environment
```
- Reasoning chỉ tồn tại trong Reasoning Model.
- Agent chỉ dispatch.
- KnowledgeTool là Tool duy nhất được Agent gọi.
- Child Tool chịu trách nhiệm theo từng domain.
- Environment là Source of Truth.
---
# Đã hoàn thành
## Agent
- Agent MVP hoàn thành.
- Dispatch request.
- Dispatch Tool.
- Trả ToolResult.
- Lặp đến Final Response.
Không chứa business logic.
---
## Model Adapter
Đã hỗ trợ nhiều implementation.
Ví dụ:
- Mock Model
- Ollama
Agent không phụ thuộc implementation của Model.
---
## Execution Runtime
Đã có Runtime cơ bản.
Bao gồm:
- Execution Session
- Execution Plan
- Execution Step
- Execution Result
- Execution Context
Runtime chỉ điều phối thực thi.
Không reasoning.
---
## KnowledgeTool
Đã hoàn thành MVP.
KnowledgeTool:
- là Tool duy nhất được Agent gọi.
- dispatch request tới Child Tool.
- trả ToolResult.
- expose capability metadata.
- không truy cập Environment trực tiếp.
- không reasoning.
---
## Child Tool
LinuxTool là Child Tool đầu tiên.
LinuxTool:
- sở hữu capability của chính mình.
- truy cập Source of Truth.
- trả ToolResult.
Child Tool là single source of truth của capability metadata.
---
## Capability Metadata
Đã hoàn thành.
Nguyên tắc hiện tại:
- Child Tool sở hữu capability.
- KnowledgeTool tổng hợp capability metadata.
- Bootstrap lấy capability metadata từ KnowledgeTool.
- PromptBuilder không duy trì static capability list.
---
## Provider
Provider vẫn là optional.
Chỉ tạo khi logic truy cập Source of Truth đủ phức tạp.
Hiện tại LinuxTool vẫn truy cập Environment trực tiếp.
---
## Testing
Repository đã có test cho:
- runtime
- model
- shared
- tool
- protocol
Test phải luôn duy trì trạng thái xanh.

# Definition of MVP
MVP hoàn thành khi:
- Reasoning Loop hoạt động ổn định.
- Agent dispatch ổn định.
- KnowledgeTool dispatch ổn định.
- Child Tool hoạt động đúng.
- Capability metadata đồng bộ.
- Provider hoạt động khi cần.
- Toàn bộ test thành công.
- Repository luôn sạch.
Cho đến thời điểm đó, mọi quyết định đều ưu tiên hoàn thành MVP thay vì bổ sung tính năng mới.
