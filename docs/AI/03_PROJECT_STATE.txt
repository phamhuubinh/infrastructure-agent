# 03 - Project State
Tài liệu này mô tả trạng thái hiện tại của project.
Mục đích:
- AI biết project đang ở giai đoạn nào.
- AI biết phạm vi cần thực hiện.
- AI tránh phát triển vượt quá MVP.
- AI không suy diễn những thành phần chưa tồn tại.
---
# Mục tiêu hiện tại
Hoàn thành MVP đầu tiên của AI Agent.
Ưu tiên:
1. Hoàn thiện luồng Reasoning.
2. Hoàn thiện Tool.
3. Hoàn thiện Agent.
4. Giữ kiến trúc đơn giản.
5. Không tối ưu sớm.
---
# Kiến trúc hiện tại
Project theo kiến trúc:
```text
User
    │
    ▼
Reasoning Model
    │
    ▼
Agent
    │
    ▼
Tool
    │
    ├────────► Stable Store
    │
    └────────► Environment
```
Model là thành phần duy nhất được reasoning.
---
# Đã hoàn thành
## Agent
Đã có Agent MVP.
Agent có khả năng:
- nhận request từ Model
- gọi Tool
- trả Tool Result
- lặp cho tới Final Response
Agent không chứa business logic.
---
## Model Adapter
Đã có abstraction để kết nối nhiều Model.
Ví dụ:
- Mock Model
- Ollama
- Model khác trong tương lai
Agent không phụ thuộc vào implementation của Model.
---
## Execution Runtime
Đã có Runtime cơ bản.
Bao gồm:
- Execution Session
- Execution Plan
- Execution Step
- Execution Result
- Execution Context
Runtime chịu trách nhiệm điều phối việc thực thi.
Không reasoning.
---
## Collector
Đã có Linux Collector.
Pipeline:
```text
collect_osquery
        │
        ▼
raw observation
        │
        ▼
knowledge builder
        │
        ▼
stable store
```
Collector hiện sử dụng osquery.
---
## Stable Store
Đã có phiên bản đầu tiên.
Stable Store lưu:
- System
- Hardware
- Network
- User
- Docker
- LXD
- Repository Information
Stable Store chỉ lưu dữ liệu thay đổi ít.
---
## Knowledge / Inventory
Đã có Tool đọc Stable Store.
Có thể:
- đọc resource
- trả dữ liệu theo resource
- phục vụ Model
---
## Test
Đã có test cho nhiều thành phần.
Bao gồm:
- schema
- runtime
- model
- shared
Project ưu tiên duy trì test hoạt động trong suốt quá trình phát triển.
---
# Đang thực hiện
Ưu tiên hiện tại:
```text
Reasoning Model
        │
        ▼
Agent
        │
        ▼
Tool
        │
        ▼
Stable Store
        │
        ▼
Tool Result
        │
        ▼
Reasoning Model
        │
        ▼
Final Response
```
Hoàn thiện toàn bộ vòng lặp reasoning.
---
# Chưa thực hiện
Các thành phần dưới đây chưa phải ưu tiên.
Không triển khai nếu chưa có yêu cầu.
- Multi Agent
- Planner
- Plugin
- Scheduler
- Runtime Store
- Embedding
- Search Engine
- Vector Database
- Cache
- Distributed Runtime
---
# Không nằm trong MVP
Hiện tại chưa cần:
- Multi Model Collaboration
- Memory Retrieval
- Long-term Conversation Memory
- Agent Communication
- Workflow Engine
- Event System
Mọi đề xuất liên quan các nội dung trên cần được xem xét lại sau khi MVP hoàn thành.
---
# Nguyên tắc phát triển
Mọi thay đổi phải phục vụ mục tiêu:
- hoàn thành MVP
- giảm độ phức tạp
- không phá kiến trúc
- không tăng coupling
Nếu một giải pháp:
- làm code phức tạp hơn
- tăng abstraction
- không giúp MVP hoàn thành
=> Không thực hiện.
---
# Thứ tự ưu tiên
Ưu tiên theo thứ tự:
1. Hoàn thiện Tool.
2. Hoàn thiện Agent.
3. Hoàn thiện luồng Agent ↔ Model.
4. Hoàn thiện Runtime.
5. Ổn định test.
6. Verification.
7. Sau khi MVP ổn định mới mở rộng.
---
# Definition of MVP
MVP được xem là hoàn thành khi:
- Agent hoạt động ổn định.
- Model có thể reasoning nhiều bước.
- Tool hoạt động đúng.
- Stable Store hoạt động đúng.
- Collector cập nhật được dữ liệu.
- Toàn bộ test thành công.
- Repository luôn ở trạng thái sạch.
Cho đến thời điểm đó, mọi quyết định đều phải ưu tiên hoàn thành MVP thay vì bổ sung tính năng mới.
