# 03 - Project State
Tài liệu này mô tả trạng thái hiện tại của project.
Mục đích:
- AI biết project đang ở giai đoạn nào.
- AI biết phạm vi cần thực hiện.
- AI tránh phát triển vượt quá MVP.
- AI không suy diễn những thành phần chưa tồn tại.
---
# Mục tiêu hiện tại
Hoàn thành MVP đầu tiên của AI Agent.
Ưu tiên:
1. Hoàn thiện luồng Reasoning.
2. Hoàn thiện Tool.
3. Hoàn thiện Agent.
4. Hoàn thiện Provider.
5. Giữ kiến trúc đơn giản.
---
# Kiến trúc hiện tại
Project theo kiến trúc:
User
    ↓
Reasoning Model
    ↓
Agent
    ↓
KnowledgeTool
    ↓
Tool con
    ↓
Provider (optional)
    ↓
Environment
Model là thành phần duy nhất được reasoning.
Environment là Source of Truth.
---
# Đã hoàn thành
## Agent
Đã có Agent MVP.
Agent có khả năng:
- nhận request từ Model
- gọi Tool
- trả Tool Result
- lặp cho tới Final Response
Agent không chứa business logic.
---
## Model Adapter
Đã có abstraction để kết nối nhiều Model.
Ví dụ:
- Mock Model
- Ollama
- Model khác trong tương lai
Agent không phụ thuộc implementation của Model.
---
## Execution Runtime
Đã có Runtime cơ bản.
Bao gồm:
- Execution Session
- Execution Plan
- Execution Step
- Execution Result
- Execution Context
Runtime chịu trách nhiệm điều phối việc thực thi.
Runtime không reasoning.
---
## Linux Tool
LinuxTool MVP đã được triển khai.
LinuxTool là Tool con đầu tiên của KnowledgeTool.
LinuxTool chịu trách nhiệm:
- truy cập Linux;
- lấy dữ liệu từ Source of Truth;
- trả ToolResult.
KnowledgeTool đang trong quá trình migrate thành Tool tổng.
LinuxProvider chưa được triển khai.
Provider sẽ chỉ được tạo khi việc truy cập Linux đủ phức tạp để cần tách riêng.
---
## Provider
Project đang chuyển sang kiến trúc Provider.
Provider là lớp truy cập Source of Truth.
Ví dụ:
- Linux Provider
- SSH Provider
- Docker Provider
- VMware Provider
- Zabbix Provider
- Grafana Provider
Mỗi Provider chỉ biết cách lấy dữ liệu từ nguồn của mình.
Không reasoning.
---
## Knowledge Tool
Đã có Tool truy cập dữ liệu.
Knowledge Tool có thể:
- nhận request từ Model;
- định tuyến request tới Tool con;
- trả ToolResult.
Knowledge Tool không trực tiếp truy cập Environment.
Knowledge Tool không trực tiếp gọi Provider.
---
## Test
Đã có test cho nhiều thành phần.
Bao gồm:
- schema
- runtime
- model
- shared
Project ưu tiên duy trì test hoạt động trong suốt quá trình phát triển.

# Stable Store
Stable Store đang được loại bỏ khỏi runtime.
Có thể tiếp tục tồn tại trong giai đoạn migration hoặc phục vụ snapshot nếu cần.
Stable Store không còn là nguồn dữ liệu chính của hệ thống.
Runtime luôn ưu tiên đọc trực tiếp từ Source of Truth thông qua Provider.

# Nguyên tắc phát triển
Mọi thay đổi phải phục vụ mục tiêu:
- hoàn thành MVP
- giảm độ phức tạp
- không phá kiến trúc
- không tăng coupling
Nếu một giải pháp:
- làm code phức tạp hơn
- tăng abstraction
- không giúp MVP hoàn thành
=> Không thực hiện.

# Thứ tự ưu tiên
Ưu tiên theo thứ tự:
1. Hoàn thiện Provider.
2. Hoàn thiện Tool.
3. Hoàn thiện Agent.
4. Hoàn thiện luồng Agent ↔ Model.
5. Hoàn thiện Runtime.
6. Ổn định test.
7. Verification.
Sau khi MVP ổn định mới mở rộng.
---
# Definition of MVP
MVP được xem là hoàn thành khi:
- Agent hoạt động ổn định.
- Model có thể reasoning nhiều bước.
- Tool hoạt động đúng.
- Provider hoạt động đúng.
- Knowledge Tool có thể truy cập Source of Truth.
- Toàn bộ test thành công.
- Repository luôn ở trạng thái sạch.
Cho đến thời điểm đó, mọi quyết định đều phải ưu tiên hoàn thành MVP thay vì bổ sung tính năng mới.
