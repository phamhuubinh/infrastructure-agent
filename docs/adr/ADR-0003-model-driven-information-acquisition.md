# ADR-0003
# Status
Accepted
---
# Context
The Agent executes actions requested by the reasoning model.
As the system evolves, multiple tools may provide information from different sources, including:
- local execution;
- remote execution;
- cached context;
- discovered knowledge;
- external systems.
Without a clear architectural boundary, the Agent or individual tools could begin deciding what information should be collected.
This would introduce reasoning into the execution layer and violate the core architectural principles.
The architecture therefore requires explicit ownership of information acquisition.
---
# Decision
The reasoning model exclusively owns information acquisition.
The reasoning model decides:
- what information is required;
- when information should be collected;
- whether previously available information is sufficient;
- whether fresh execution is required.
The Agent never decides what information should be collected.
The Agent never performs implicit execution.
Every execution performed by the Agent must originate from an explicit Action generated by the reasoning model.
Tools execute only the requested operation.
Tools may internally optimize execution using implementation-specific mechanisms such as cached sessions, persistent connections, or local configuration.
These implementation details remain invisible to the reasoning model.
Tools never decide to collect additional information beyond the requested operation.
---
# Stable Information
Information that changes infrequently may be retrieved through dedicated tools instead of repeated execution.
Examples include:
- hardware configuration;
- operating system information;
- SSH host configuration;
- discovered capabilities;
- static environment metadata.
Whether this information should be reused is always determined by the reasoning model.
---
# Dynamic Information
Information that changes during system operation shall not be assumed to remain valid.
Examples include:
- CPU utilization;
- memory utilization;
- running processes;
- service status;
- container state;
- network state;
- active connections.
Fresh information shall only be collected when explicitly requested by the reasoning model.
---
# Consequences
The reasoning model remains the only component capable of deciding what information is valuable.
The Agent remains deterministic.
Tools remain execution components rather than reasoning components.
Execution cost is reduced because only explicitly requested information is collected.
Prompt size is minimized because unnecessary observations are never generated.
The architecture remains model-driven, execution-driven, and model-agnostic.
