# Execution Step Specification
---
# Purpose
Execution Step represents a single executable operation generated by the reasoning model.
Execution Step defines one atomic execution task for the Agent.
Execution Step never performs execution.
Execution Step never performs reasoning.
---
# Responsibilities
Execution Step is responsible for:
* representing one executable operation;
* preserving execution order within an Execution Plan;
* carrying execution metadata;
* remaining immutable after creation.
Execution Step must never:
* execute itself;
* perform reasoning;
* modify itself after creation;
* interpret execution results.
---
# Ownership
Execution Step belongs exclusively to the reasoning model.
Execution Step is consumed by the Agent.
Execution Step is never modified by the Agent.
---
# Inputs
Execution Step receives no runtime input.
Execution Step is generated entirely by the reasoning model.
---
# Outputs
Execution Step provides one executable operation to the Agent.
Execution Step itself produces no execution results.
---
# Lifecycle
The Execution Step lifecycle consists of:
1. Generated by the reasoning model.
2. Added to an Execution Plan.
3. Executed by the Agent.
4. Completed.
5. Archived or discarded according to higher-level policies.
---
# Architectural Constraints
Execution Step shall remain:
* immutable;
* deterministic;
* model-generated;
* execution-only;
* independent of Runtime implementation;
* independent of execution results.
Execution Step shall not contain execution state.
Execution Step shall not contain execution results.
Execution Step shall not perform planning after creation.
