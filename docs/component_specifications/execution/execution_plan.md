# Execution Plan Specification
---
# Purpose
Execution Plan represents a complete execution workflow generated by the reasoning model.
Execution Plan defines what the Agent shall execute.
Execution Plan never performs execution.
Execution Plan never performs reasoning.
---
# Responsibilities
Execution Plan is responsible for:
* representing a complete execution workflow;
* defining execution steps;
* preserving execution order;
* carrying execution metadata;
* remaining immutable after creation.
Execution Plan must never:
* execute commands;
* perform reasoning;
* modify itself after creation;
* analyze execution results.
---
# Ownership
Execution Plan belongs exclusively to the reasoning model.
Execution Plan is consumed by the Agent.
Execution Plan is never modified by the Agent.
---
# Inputs
Execution Plan receives no runtime input.
Execution Plan is generated entirely by the reasoning model.
---
# Outputs
Execution Plan provides execution steps to the Agent.
Execution Plan itself produces no execution results.
---
# Lifecycle
The Execution Plan lifecycle consists of:
1. Generated by the reasoning model.
2. Delivered to the Agent.
3. Executed by the Agent.
4. Completed.
5. Archived or discarded according to higher-level policies.
---
# Architectural Constraints
Execution Plan shall remain:
* immutable;
* deterministic;
* model-generated;
* execution-only;
* independent of Runtime implementation;
* independent of execution results.
Execution Plan shall not contain execution state.
Execution Plan shall not contain execution results.
Execution Plan shall not perform planning after creation.
